Hi my name is here even if this function is run in a separate isolate the function getAssetListPaged need to communicate with I/O Flutter engine on the main isolate so the idea of running that funcion into a subTask and then waif for all the subTask to complete was a huge bottle neck because all the subTask weren't actually run immidiatly. They are all triggered by the Future.wait that, with very large folders has to create a lot of messages and have to send them to the I/O flutter engine that became saturated by requests and more important ones like the one from the ui are put into the queue and that cause a significant lag into the app. So avoid to use subtask even if it was making the function faster because it was blocking the ui.
